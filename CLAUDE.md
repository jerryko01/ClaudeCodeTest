This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository overview

This repo contains a single self-contained web game: `tic-tac-toe.html`. There is no build system, package manager, bundler, linter, or test suite — the entire game (markup, CSS, and JavaScript) lives in one HTML file with no external dependencies.

## Running the game

Open `tic-tac-toe.html` directly in a browser (double-click it, or `start tic-tac-toe.html` on Windows). There is no dev server or build step — edits to the file are reflected immediately on reload.

## Architecture

Everything is in `tic-tac-toe.html`, organized top to bottom as:

- **`<style>`** — theme variables under `:root` (with a `prefers-color-scheme: dark` override) drive all colors; component styles follow (`.app`, `.board`, `.cell`, `.reset`, `.score`).
- **`<body>`** — static markup for the status line, the empty `#board` container (cells are generated by JS, not hardcoded), the reset button, and the win/draw scoreboard.
- **`<script>`** — a single IIFE holding all game state and logic:
  - `board` (array of 9 cells), `current` (whose turn), `gameOver`, and `scores` are the only state.
  - `checkWinner` / `isDraw` are pure functions over a board array (used both for the live board and, previously, for move simulation — keep this pattern if re-adding any kind of look-ahead logic).
  - `render()` is the single place that syncs the DOM to state; call it after any state mutation rather than patching the DOM directly elsewhere.
  - `afterMove()` is the central turn-resolution step (win/draw check, score update, turn switch) — route all move logic through it so win/draw/score handling stays in one place.
